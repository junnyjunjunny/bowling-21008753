# Plan: 스페어가 있는 경우 보너스 점수 계산

## 목표 (RED)

한 프레임에서 스페어(두 투구 합이 10)가 발생했을 때, 다음 투구 1회가 해당 프레임에 보너스로 더해지는지 검증하는 테스트를 작성한다.

## 작성할 테스트

파일: `tests/test_game.py`

```python
def test_one_spare_adds_next_roll_as_bonus():
    game = Game()

    game.roll(3)
    game.roll(7)  # 1프레임: 스페어 (3+7=10)
    game.roll(4)  # 보너스 투구 + 2프레임의 첫 투구
    game.roll(0)  # 2프레임의 두 번째 투구

    for _ in range(16):  # 3~10프레임: 전부 거터
        game.roll(0)

    assert game.score() == 18
```

- 프레임별 기대 점수: 1프레임 = 3+7+4(보너스) = 14, 2프레임 = 4+0 = 4, 3~10프레임 = 0. 합계 18.
- 현재 구현(`sum(self.rolls)`)은 보너스를 중복으로 더해주지 않으므로 이 테스트는 **실제로 실패**할 것으로 예상된다 (14가 아닌 18을 기대하지만 단순 합계는 3+7+4+0*17=14를 반환). → 진짜 RED.

## 예상되는 최소 구현 범위 (GREEN에서 진행 예정)

파일: `src/game.py`

- `score()`를 "전체 투구의 단순 합계" 방식에서 **프레임 단위 순회 + 스페어 보너스** 방식으로 변경한다.
- 각 프레임(현재는 스트라이크 없는 케이스만 고려)에서:
  - 두 투구의 합이 10(스페어)이면, 프레임 점수 = 10 + 다음 투구 1회.
  - 그렇지 않으면, 프레임 점수 = 두 투구의 합.
- 10개 프레임을 순회하며 프레임 점수를 누적한다.
- 스트라이크 처리, 10번째 프레임의 추가 투구 처리는 이번 사이클에서 다루지 않는다 (다음 사이클들에서 별도로 확장).
  - 단, 이번 테스트 데이터에는 스트라이크가 없으므로 스트라이크 분기를 아직 만들 필요는 없다. 프레임 인덱스 이동 시 "이 투구가 스트라이크였는지"는 고려하지 않고, 항상 프레임당 2개 투구를 소비하는 방식으로 구현한다 (기존 테스트 두 개는 모두 프레임당 2투구이므로 회귀 없음).
