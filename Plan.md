# Plan: 모든 투구가 1핀인 경우 총점 20

## 목표 (RED)

`Game` 클래스에 20번 연속으로 1핀을 투구했을 때, `score()`가 20을 반환하는지 검증하는 테스트를 작성한다.

## 작성할 테스트

파일: `tests/test_game.py`

```python
def test_all_ones_scores_twenty():
    game = Game()

    for _ in range(20):
        game.roll(1)

    assert game.score() == 20
```

- 기존 `test_all_gutters_scores_zero`와 마찬가지로 10프레임 × 2투구 = 20번의 `roll(1)` 호출.
- 스페어(프레임당 합이 10)나 스트라이크가 발생하지 않는 값(1)을 사용하므로, 이번에도 보너스 계산 로직이 개입하지 않는다.
- 현재 `Game.score()` 구현(모든 투구의 단순 합계)이 이미 이 케이스를 통과시킬 것으로 예상된다 — 즉 이번 사이클은 기존 구현이 "단순 합계로 충분하다"는 가정을 재확인하는 회귀 테스트 성격에 가깝다.

## 예상되는 최소 구현 범위 (GREEN에서 진행 예정)

- `src/game.py`는 변경이 필요 없을 가능성이 높다 (기존 `sum(self.rolls)` 구현으로 통과 예상).
- 테스트 실행 결과 실제로 통과하면 그대로 GREEN 확인만 하고 넘어간다.
- 만약 예상과 달리 실패한다면, 그 실패 원인을 확인한 뒤 이 Plan을 다시 검토한다 (수정된 구현 범위를 사람 파트너와 재합의).
