# Plan: 퍼펙트 게임(전부 스트라이크) → 총점 300

## 목표 (RED)

12번 연속 스트라이크(10핀)를 투구했을 때, `score()`가 300을 반환하는지 검증하는 테스트를 작성한다.

## 작성할 테스트

파일: `tests/test_game.py`

```python
def test_perfect_game_scores_three_hundred():
    game = Game()

    for _ in range(12):
        game.roll(10)

    assert game.score() == 300
```

- 10프레임 모두 스트라이크 + 10번째 프레임의 추가 투구 2회(스트라이크 후 보너스 투구) = 총 12회 투구.
- 사전 확인 결과, 현재 스트라이크 분기 구현(`roll_index`를 1씩 증가시키며 `rolls[roll_index+1]`, `rolls[roll_index+2]`를 보너스로 더하는 방식)이 이 입력에 대해 **이미 300을 계산한다** (프레임 10에서 `roll_index=9`일 때 `rolls[10]`, `rolls[11]`이 모두 존재하므로 인덱스 초과가 발생하지 않음).
- 즉 이번 사이클도 `test_all_ones_scores_twenty`와 마찬가지로, 새 동작을 추가하는 것이 아니라 기존 구현이 극단적인 케이스(전부 스트라이크)에서도 올바르다는 것을 확인하는 회귀 테스트 성격이다. → 테스트는 즉시 통과할 것으로 예상되며, 이는 이번 케이스가 이미 구현된 로직의 범위 안에 있기 때문이지 테스트가 잘못되어서가 아니다.
